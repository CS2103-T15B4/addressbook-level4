Project: Bluebird

Bluebird is a desktop based organisational tool for the working parent on the go. Bluebird is suitable for a variety of day-to-day management tasks, including managing appointments for both yourself and your child, and also organising the contact information from your Address Book. The user interacts with it using a CLI, and it has a GUI created with JavaFX. It is written in Java, and has about 15 kLoC.

This project portfolio serves to provide an insight into how Azizi Azfar has contributed to project Bluebird. Included in this portfolio are the enhancements, features, tests and proposed enhancements that Azizi Azfar has contribued to the project.

Enhancement Added: Multiple Delete

External behavior

Start of Extract [from: User Guide]

Delete person(s) : delete

Deletes the specified person(s) from Bluebird’s address book.
Format: delete EXAMPLEINDEX1 EXAMPLEINDEX2 INDEX RANGE

Examples:

  • delete 2 3 4~7
    Deletes the 2nd, 3rd and the 4th to 7th (both inclusive) person in Bluebird’s address book.

  • find Betsy
    delete 1
    Deletes the 1st person in the results of the find command.

  • list
    delete 2~5
    Deletes 2nd to 5th person (both inclusive) in Bluebird’s address book.

  • Deletes the person at the specified INDEX or the specified INDEX RANGE.

  • The index refers to the index number shown in the most recent listing.

  • Acceptable inputs:

    • The input can be a single positive integer 1, 2, 3, …​

    • The input can be multiple positive integers with a space in between 1 2 3 …​

    • The input can be a range of positive integers, denoted by a "~" in between the minimum and maximum index (both inclusive) 1~4, 2~7, …​

End of Extract

Justification

Users may need to delete multiple contacts from the Bluebird application and thus, this feature will make it convenient for the users to do so.

Implementation

Start of Extract [from: Developer Guide]

Enhanced Delete

The enhanced delete feature allows for multiple indices to be accepted by the parser. This is done by splitting the inputs into multiple string tokens, each containing the index of the person to be deleted. A list is then used to store all the indices of the personsToDelete. This enhancement is then achieved by iterating through the list and deleting the desired persons using the ModelManager. The enhanced delete also allows for the user to input a range of indices of persons to be deleted.

Design Considerations

Aspect: Defining how accurate the delete command will be
Alternative 1 (current choice): Indices are parsed into a list containing all the index of persons to be deleted.
A loop is then used to iterate through the list to delete the persons requested by the user.
Pros: If the index is valid (within range), the user can delete as many persons as they choose.
Cons: The use of a loop will increase the time for the code to run.
Alternative 2: The indices are deleted one by one, without using a list.
Pros: If the index is valid (within range), the user can delete as many persons as they choose.
Cons: It has to display the result repeatedly after every every person is deleted successfully.

End of Extract

Enhancement Added: Add fields individually

External behavior

Start of Extract [from: User Guide]

Adds a person: add

Adds a person’s contact information to Bluebird’s address book based on the fields declared by the user.
Format: add n/NAME [p/PHONE_NUMBER] [e/EMAIL] [a/ADDRESS] [t/TAG] …​

Examples:

  • add n/John Doe p/98765432 e/johnd@example.com a/John street, block 123, #01-01

  • add n/Johnny Dow

  • add n/Carl Green p/91234765

  • add n/Betsy Crowe t/friend e/betsycrowe@example.com a/Newgate Prison p/1234567 t/criminal

  • A person can have any number of tags (including 0).

  • Only n/NAME field has to be added, the rest are optional.

  • Fields can be updated in the future using the EDIT command.

End of Extract

Justification

Users may only intend to add the name and phone number field in Bluebird. Thus, with the enhanced functionality of the add command, users can do so without any difficulty.

Implementation

Start of Extract [from: Developer Guide]

Enhanced Add

The enhanced add feature no longer requires the user to fill in every field as it now only requires the n/NAME field to be filled. This feature is done by using a constructor to initialize the default field as empty. This is implemented using an additional parameter in the ParserUtil class to determine whether the default constructor for an empty field should be called.

An extract of the code from ParserUtil class shows how this is being implemented. The command parameter determines whether or not a field is required in the parser. If the parser is called from the add command, it will call a default constructor where the parsed field is empty.

public static Optional<Name> parseName(Optional<String> name, String command) throws IllegalValueException {
        requireNonNull(name);
        if (command.equals("add")) {
            return name.isPresent() ? Optional.of(new Name(name.get())) : Optional.of(new Name(0));
        } else {
            return name.isPresent() ? Optional.of(new Name(name.get())) : Optional.empty();
        }
    }

Design Considerations

Aspect: Implementing an option to accept and create the empty fields
Alternative 1 (current choice): By using a command parameter in the ParserUtil class
Pros: This will allow flexibility in implementing, or calling the necessary constructors in the respective person field classes
Cons: There will be a need for overloading of constructors in the classes for the person fields
Alternative 2: By having a new class to initialise empty fields for person added, similar to the EditPersonDescriptor class in the Edit command
Pros: This is the most comprehensive way to initialise fields without the need of new constructors in the classes for the respective fields
Cons: The code will be unnecessarily lengthy and may cause features to break

End of Extract

Enhancement Proposed: Add multiple fields

Users may add multiple instances for any of the Person fields in Bluebird. For example, users can now add multiple phone numbers for any of their contacts, and thus make it much more convenient for them to update their friends particulars.

Enhancement Proposed: Deleting persons by their tags

Users may delete multiple persons in one go by selecting a tag to be deleted. Thus, users will be able to delete anyone associated with that tag when the need arises.
